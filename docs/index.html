Why

A re-shareable copy of a calendar you can't reshare

Subscribed work feeds, read-only team calendars, accounts you don't control — cal-mirror makes an editable copy on a calendar you do, then lets macOS sync it wherever that account goes.

↔️

Any → any

Mirror any calendar into any other. Configure as many pairs as you like.

♻️

Idempotent & one-way

Re-runs never duplicate. The source is authoritative and is never written to.

🔁

Recurring-safe

Recurring events are expanded into occurrences — no RRULE translation, exceptions resolved.

🛡️

Non-destructive

Each pair tags only its own copies, so shared destinations and hand-added events stay safe.

📊

Menu-bar UI

Health at a glance, Sync now, pause, interval, and pickers to add or edit pairs.

🔑

No credentials

Works through EventKit on your Mac. Server sync is macOS's job, not another password.

Glanceable, then out of the way

The icon reflects the worst mirror at a glance:

  • ✓ synced recently
  • ⚠︎ stale — last run older than twice the interval
  • ✗ error · ⏸ paused

Manage mirrors… opens a native window where you add pairs with dropdowns of your actual calendars — destinations limited to writable ones. An optional all-day heartbeat banner writes “last synced” right into the destination calendar as a liveness signal.

How it works

Three moving parts

A scheduled engine does the sync; a menu-bar app is a thin window onto it; two small JSON files carry state.

1

Read & expand

Each interval, the engine reads the source over a rolling window and expands recurring events into occurrences.

2

Upsert & tag

It creates/updates/deletes copies in the destination, each tagged so its sweep only touches its own events.

3

Sync out

macOS pushes the destination up to iCloud / CalDAV / Exchange automatically — no credentials stored anywhere.

Get it

Download, or build from source

Grab the notarized apps from the latest release, or clone and run the installer.

# From source (macOS 14+, Xcode Command Line Tools)
git clone https://github.com/mattbaylor/cal-mirror.git
cd cal-mirror
./install.sh        # builds both apps, installs the LaunchAgents

# Then grant Calendar access when prompted, and add a pair from the menu bar.
